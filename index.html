🔐 Privacy Policy

✓ Zero Personal Data ✓ AES-256 Encryption ✓ No Cross-User Access ✓ No Analytics ✓ Zero-Knowledge Storage

1. Who We Are

PromptForge is a free, browser-based AI prompt tool. We operate on a zero personal data collection principle with zero-knowledge architecture — meaning even we cannot read your data.

2. What We Do NOT Collect

  • Your name, email address, or any contact info
  • Your IP address or geographic location
  • The topics or prompts you generate
  • Browsing history, device fingerprints, or behavioural data
  • Payment information (the tool is free)

3. Anonymous Visitor Counting

If you accept cookies, we store a randomly generated anonymous device ID in your browser. This is used only to count unique visitors. It contains no personal information and is never shared with third parties.

4. Your Prompt History — Zero-Knowledge Encrypted

Your prompt history is encrypted in your browser using AES-256-GCM with a unique key derived from your anonymous device ID. This means:

  • Only your device can decrypt and read your history
  • Other users cannot see your prompts — ever
  • We (the developers) cannot read your history — even if we wanted to
  • Server operators cannot read your history
  • The encryption key never leaves your browser

5. Data Isolation

Each user's stored data is keyed to their unique anonymous device ID. There is no shared history, no admin view, and no mechanism to access another user's data.

6. Third-Party Services

PromptForge loads Google Fonts for typography only. No analytics (Google Analytics, Mixpanel etc.), no advertising networks, no tracking pixels are used anywhere.

7. Children's Privacy

This service is intended for users aged 13 and over. We collect no data from any user, including minors.

8. Your Rights

Since we hold no personal data, there is nothing to request or delete. You can clear all stored data at any time using the "Clear History" button in the app, or via your browser's site data settings.

🔒 Security Architecture

AES-256-GCM CSP Headers XSS Protected Rate Limited Zero-Knowledge

Encryption — AES-256-GCM

All prompt history is encrypted using AES-256-GCM via the browser's native Web Crypto API before any storage. A unique encryption key is derived from your anonymous device ID using PBKDF2 with 100,000 iterations and a SHA-256 hash. The key never leaves your browser.

Data Isolation

Every user's data is stored under a unique key namespace. There is no shared state, no cross-user lookup, and no admin override. Even with full database access, stored data cannot be read without the device-specific encryption key.

Content Security Policy (CSP)

A strict Content Security Policy is enforced in the HTML meta tag, blocking: inline script injection, external script loading from unknown sources, iframes (clickjacking), object embeds, and data exfiltration via connect-src: none.

XSS (Cross-Site Scripting) Protection

All user input is sanitized using HTML entity encoding before any DOM insertion. Input lengths are capped. The app never uses innerHTML with user content.

Rate Limiting

Prompt generation is rate-limited to 20 requests per 60 seconds per device, enforced in the browser. This prevents automated abuse.

Input Validation Firewall

A multi-pattern regex filter screens all inputs for harmful content before processing. Blocked patterns include: weapons, CSAM, self-harm, jailbreaking attempts, hacking, hate speech, and more.

No External Data Transmission

No user input, topic, or generated prompt is ever transmitted to any server. All processing is 100% local in your browser. The CSP connect-src: none directive enforces this at the browser level.

Anti-Clickjacking

The X-Frame-Options: DENY meta directive prevents the app from being embedded in iframes, blocking UI redressing attacks.
